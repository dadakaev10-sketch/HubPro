15 · Sicherheit

Security Architecture

🔐 Firebase Auth

Email/Password + Session-Tokens. Demo-Mode via sessionStorage für Testzwecke.

🛡️ Firestore Rules

Role-based Security: Kunden können nur eigene Inhalte sehen + Kommentare schreiben.

🔑 API Keys

Alle Secrets in .env — niemals im Git-Repo. VITE_* Prefix für Client-seitige Variablen.

🚫 Data Isolation

Kunde A sieht nie Inhalte von Kunde B. clientName-basierte Filterung serverseitig.

// Firestore Security — Kunden dürfen nur readBy + status ändern
allow update: if request.auth != null
  && request.resource.data.diff(resource.data)
    .affectedKeys().hasOnly(['comments', 'status']);
